Creating custom components (Standalone Project)

Custom Component (Standalone Project)

  1. Create a new file
    • For example: header.component.ts

    or

    • Use command: ng generate component header
  2. Import the required module 
    import { Component } from '@angular/core';
  3. Use the @Component decorator
    • This decorator provides metadata for the component, such as its selector, template, and styles.
  4. Define a selector
    • The selector tells Angular which elements on the screen should be controlled or replaced by our custom component.
    • The convention is to use at least two words separated by a dash (Example: app-header).
    • Using a single-word selector may conflict with built-in HTML elements.
  5. Specify a template
    • The template defines the HTML structure of the component.
    • We can define it using:
    • A string (inline template) where we write HTML directly.
    • A separate file using templateUrl, where we provide the file path.
  6. Set the standalone property
    • Depending on the Angular version:
    • Angular 19+: The standalone property is automatically set to true.
    • Earlier versions: You must manually set standalone: true.
    • For module-based components, set it to false.
  7. Import the component into app.component.ts
    • To use the HeaderComponent, import it in app.component.ts and include it in the template. 
      •                  
                  import { Component } from '@angular/core';
                  import { HeaderComponent } from './header.component';

                  @Component({
                  selector: 'app-root',
                  standalone: true,
                  imports: [HeaderComponent],
                  templateUrl: './app.component.html',
                  styleUrls: ['./app.component.css']
                  })
  8. Some Angular Features
    • Inside a component we can create variables and assign a value to it without using var, let and const keywords
    • We a create regular variable outside the class and do regular javascript there
      •           
            import { Component } from '@angular/core';
            import { DUMMY_USERS } from '../../dummy-users';

            const randomIndex = Math.floor(Math.random() * DUMMY_USERS.length);

            @Component({
            selector: 'app-user',
            standalone: true,
            templateUrl: './user.component.html',
            styleUrl: './user.component.css',
            })
            export class UserComponent {
           selectedUser = DUMMY_USERS[randomIndex];
           }
    • We can also import a normal .ts file and use there values in our component
      •     
      export const DUMMY_USERS = [
        {
         id: 'u1',
         name: 'Jasmine Washington',
         avatar: 'user-1.jpg',
       },
       {
         id: 'u2',

     ];
    • String interpolation – Display dynamic content. If we add private before the property, we can access it only inside the class. If we do not add public, it will be public by default. We can use other JavaScript expressions inside the interpolation {{ }} 
      •   
    export class UserComponent {
     private selectedUser = DUMMY_USERS[randomIndex];
  }
  

      
<p>{{ selectedUser.name }}</p>
    • Property Binding - Dynamic usage of html attribute. We can concatenate the src to get dynamic path 
      • 
<img [src]="{{ selectedUser.avatar }}" alt="user-image" />
  9. Attribute Binding
    • In Property Binding, Angular binds values to DOM properties, not attributes. For example: 
      • 
<img [src]="someSrc">
    • Here, src is bound to the DOM property, not the HTML attribute.
    • However, some attributes (like ARIA attributes) don’t have matching DOM properties. In such cases, Attribute Binding is used: 
      • 
<div 
  role="progressbar" 
  [attr.aria-valuenow]="currentVal" 
  [attr.aria-valuemax]="maxVal">
</div>
    • Adding attr. before an attribute tells Angular to bind the **HTML attribute instead of a property. This is useful for ARIA, colspan, tabindex, and other attributes without corresponding properties.

  11. Signals
    • A Signal is an object that stores a value (any type of value, including nested objects) There are two approaches for update state
      • Option 01:
    • Relying on Zone.js & Angular’s changes detection mechanism
    • Works automatically, no special instruction mechanism
    • Supports since Angular 2
      • Option 02:
    • Using Signals to notify angular about value changes & required Ui updates
    • Requires usage & special “signal” instruction & code
    • Supported since angular 16
      • How to use Signals 
      
import { Component, signal } from '@angular/core';
    • We need to call the signal to access the property inside it
      • 
selectedUser = signal(DUMMY_USERS[randomIndex]);

imagePath = computed(() => './assets/users/' + this.selectedUser().avatar);

get imagePath() {
  return './assets/users/' + this.selectedUser().avatar;
}

onSelectUser() {
  const randomIndex = Math.floor(Math.random() * DUMMY_USERS.length);
  this.selectedUser = signal(DUMMY_USERS[randomIndex]);
}
